## 結論
コンピュータ同士が通信するときの「共通ルール（約束事）」。

## 噛み砕いた説明
人間で例えるなら：
- 日本語同士 → 会話できる
- 日本語 × 英語 → 通じない

これと同じで、同じプロトコルを使っていないと通信できない。

## プロトコルで決めていること
通信する際には以下の内容を厳密に決めている。
- データの送り方
- データの区切り方
- エラーが起きたときの対応方法
- 誰宛てのデータか（識別方法）
- 通信を開始・終了する手順

→ これを決めていないと、通信はカオスになる。

## 身近なプロトコル例
| プロトコル     | 何をするものか      |
| --------- | ------------ |
| **HTTP**  | Webページを見る    |
| **HTTPS** | 暗号化されたWeb通信  |
| **TCP**   | 信頼性のある通信の土台  |
| **IP**    | 宛先を指定する      |
| **FTP**   | ファイル転送       |
| **SMTP**  | メール送信        |
| **RDP**   | リモートデスクトップ操作 |
| **SSH**   | リモートでコマンド操作  |

## なぜ重要なのか
- プロトコルがあることによって、メーカーやOSが違っても通信できる。
- プロトコルがないと、どう解釈するかで全部ずれる。

## RDPでの考え方
昨日RDPについてまとめたので繋げると、

RDP = 「画面・キーボード・マウス」を遠隔でやりとりするための通信ルール

「RDP通信する」 = 「RDPというルールに従ってリモート操作する」
