## 1. データ型の全体像

Javaのデータ型は大きく2種類に分類される。

- プリミティブ型（基本型）
- 参照型

この違いは「値を持つか」「アドレスを持つか」の違いである。


## 2. プリミティブ型

### ■ 一覧

| 型 | ビット数 | 範囲 |
|----|----------|------|
| byte | 8bit | -128〜127 |
| short | 16bit | -32768〜32767 |
| int | 32bit | -2^31〜2^31-1 |
| long | 64bit | -2^63〜2^63-1 |
| float | 32bit | 浮動小数 |
| double | 64bit | 浮動小数 |
| char | 16bit | 0〜65535（UTF-16） |
| boolean | JVM依存（概念上1bit） | true/false |


### ■ 特徴

- 値そのものを保持
- nullを持てない
- 固定長ビットで設計されている
- 高速
- メソッドを持たない

## 3. 固定長データとは何か

プリミティブ型は「ビット数が固定」されている。

```例：intは32bit
00000000 00000000 00000000 00001010
```
これは10を表す。（工業高校得意の2進数）

コンピュータは全て「0と1」で計算している。


### ■ なぜ固定長なのか

- CPUが高速処理できる
- メモリ管理が容易
- 直接演算可能

可変長にすると演算が遅くなる。

## 4. 符号付き整数と2の補数

intは32bitのうち

- 1bit → 符号
- 31bit → 数値

```最大値
01111111 11111111 11111111 11111111
= 2^31 - 1
= 2147483647
```

### ■ 負数の表現（2の補数）

```-1の例
1をビット反転＋1

00000000 00000000 00000000 00000001
↓反転
11111111 11111111 11111111 11111110
↓+1
11111111 11111111 11111111 11111111
```

これが -1。


## 5. オーバーフロー

```最大値 + 1 をすると

01111111 11111111 11111111 11111111
+1

10000000 00000000 00000000 00000000
```
符号ビットが1になる。

→ -2147483648

Javaはエラーを出さない。
これをオーバーフローという。

## 6. 型変換

### ■ 暗黙的変換（拡大変換）

```
byte → short → int → long → float → double
```

小さい箱から大きい箱へ入れる。安全。

### ■ 明示的変換（縮小変換）

```java
int x = (int) 3.14;
```

情報が失われる可能性あり。

## 7. 変数の初期値

### ■ ローカル変数

```java
void test() {
    int x;
    System.out.println(x); // コンパイルエラー
}
```

自動初期化されない。

理由：未定義値の使用を防ぐため。

### ■ インスタンス変数 / static変数

| 型       | 初期値   |
| ------- | ----- |
| int     | 0     |
| boolean | false |
| 参照型     | null  |

## 8. メモリ構造（JVM視点）

Javaのメモリは主に3領域。

### ■ スタック

- メソッド呼び出しごとに生成
- ローカル変数を保存
- LIFO構造
- メソッド終了で自動破棄

### ■ ヒープ

- newしたオブジェクトを保存
- GC対象
- 全スレッド共有

### ■ メソッドエリア

- クラス情報
- static変数
- 定数プール

## 9. 参照型

### ■ 特徴

- オブジェクトのアドレスを保持
- nullを持てる
- メソッドを持てる
- ヒープ上の実体を指す

### ■ 代表例

- String
- 配列
- List
- Map
- Integer
- 自作クラス
- Exception

基本的に「newするもの」は参照型。

## 10. プリミティブ vs 参照型

### ■ プリミティブ

値を直接保持
```java
int a = 10;
int b = a;
```
→ 値コピー

### ■ 参照型

```java
List l1 = new ArrayList();
List l2 = l1;
```
→ 同じオブジェクトを見る

## 11. ラッパークラス

### ■ 代表例

- Integer
- Long
- Double
- Boolean

### ■ 内部構造（例：Integer）

```java
public final class Integer {
    private final int value;
}
```

intを内部に持つオブジェクト。

## 12. オートボクシング

```java
int a = 10;
Integer b = a;
```

内部では：
```java
Integer b = Integer.valueOf(10);
```

## 13. アンボクシング

```java
Integer a = 10;
int b = a;
```
内部では：
```java
int b = a.intValue();
```

## 14. Integerキャッシュ

-128〜127は事前生成。
```java
Integer a = 100;
Integer b = 100;
```

→ 同一参照
```java
Integer a = 200;
Integer b = 200;
```

→ 別オブジェクト

- == は参照比較
- equals は値比較

## 15. 実務で重要なポイント

### ① DBとのnull問題

- intではNULLを表現できない。
- 未設定と0の区別ができない。

実務では：

```java
Integer age;
```

を使用することが多い。

### ② パフォーマンス問題

ループでIntegerを使うと

- ボクシング
- アンボクシング
- GC増加

高負荷処理ではプリミティブを使う。

### ③ 参照の副作用

参照型はアドレスコピー。

意図せず値が変更されることがある。

## 16. まとめ

- プリミティブは固定長ビットデータ
- 参照型はヒープ上のオブジェクトを指す
- intは32bit符号付き整数（2の補数）
- オーバーフローはエラーにならない
- メモリはスタックとヒープに分かれる
- ラッパークラスはオブジェクトとして振る舞うために存在する
