## 1. 演算子とは何か

演算子とは「値に対して処理を行うための記号」である。

Javaの演算子は内部的には「バイトコード命令」に変換される。

```例
a + b → iadd（intの場合）
```

つまり演算子は単なる記号ではなく、CPU命令に直結する。

## 2. 算術演算子

### ■ 種類

```java
+  -  *  /  %
```

### ■ 整数除算の注意

```java
System.out.println(5 / 2); // 2
```
整数同士の割り算は小数切り捨て。

### ■ 浮動小数との混在

```java
System.out.println(5 / 2.0); // 2.5
```

doubleへ暗黙変換される。

### ■ オーバーフロー

```java
int x = 2147483647;
x += 1;
System.out.println(x); // -2147483648
```

エラーは出ずにそのまま計算される。

**Javaはチェックしない**

## 3. 代入演算子

### ■ 基本

```java
=
+=
-=
*=
/=
```

### ■ 重要：暗黙キャストが発生する

```java
byte b = 10;
b += 1; // OK
b = b + 1; // コンパイルエラー
```

理由：
- b + 1 は intになる。
- しかし += は内部でキャストされる。

内部的には：
```java
b = (byte)(b + 1);
```

## 4. 比較演算子

```java
== != > < >= <=
```

### ■ プリミティブ比較

値比較。

### ■ 参照型の ==

参照比較（アドレス）

```java
Integer a = 100;
Integer b = 100;
System.out.println(a == b); // true（キャッシュ範囲）

Integer x = 200;
Integer y = 200;
System.out.println(x == y); // false
```

### ■ equalsとの違い

- == → 参照
- equals → 値（実装次第）

## 5. 論理演算子

### ■ 種類

```java
&& || !
```

### ■ 短絡評価（超重要）

```java
if(a != null && a.length() > 0)
```

左がfalseなら右は評価しない。

これがNullPointerException防止に重要。

### ■ & と && の違い

- & → 常に両方評価
- && → 左がfalseなら右評価しない

## 6. ビット演算子

### ■ 種類

```java
& | ^ ~ << >> >>>
```

## 7. ビットAND

```java
5 & 3
```

- 5 → 0101
- 3 → 0011

結果 → 0001（1）

## 8. ビットOR

```java
5 | 3
```

- 5 → 0101
- 3 → 0011

結果 → 0111（7）

## 9. 排他的論理和（XOR）

```java
5 ^ 3
```

- 0101
- 0011

→ 0110（6）

## 10. 反転（NOT）

```java
~5
```

- 5 → 00000000 00000000 00000000 00000101
- 反転 → 11111111 11111111 11111111 11111010

これは -6

理由：2の補数表現。

## 11. シフト演算子

### ■ 左シフト <<

```java
5 << 1
```

- 0101
- 一つ左にズレる

→ 1010（10）

2倍になる。

### ■ 右シフト >>

符号を維持する。

### ■ 論理右シフト >>>

符号無視。

負数で挙動が変わる。
