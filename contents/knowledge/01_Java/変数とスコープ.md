## 1. 変数とは何か

変数とは「値」または「参照」を保持するための名前付き領域である。

Javaでは、変数は以下の3種類に分類される。

- ローカル変数
- インスタンス変数
- static変数（クラス変数）

この違いは「どこに保存されるか」「いつ生成されるか」「いつ破棄されるか」にある。

## 2. ローカル変数

### ■ 定義

メソッド内・ブロック内で宣言される変数。

```java
void test() {
    int x = 10;
}
```

### ■ 特徴

- スタックに保存される
- メソッド呼び出し時に生成
- メソッド終了時に破棄
- 自動初期化されない
- finalでなくても実質final制約あり（ラムダ）

### ■ 初期化されない理由

```java
int x;
System.out.println(x); // コンパイルエラー
```

JVMはローカル変数を自動初期化しない。

理由：

- 未定義値の使用防止
- バグの早期発見
- パフォーマンス最適化（不要なゼロ埋めを避ける）

### ■ スタック構造

メソッド呼び出しごとに「スタックフレーム」が作られる。

```java
main()
  └─ methodA()
      └─ methodB()
```
LIFO構造（後入れ先出し）

## 3. インスタンス変数

### ■ 定義

クラス内、メソッド外に宣言される変数。

```java
class User {
    int age;
}
```

### ■ 特徴

- ヒープ上のオブジェクト内部に存在
- オブジェクト生成時に作られる
- デフォルト値で初期化される
- オブジェクトがGCされると消える

### ■ デフォルト値

| 型       | 初期値   |
| ------- | ----- |
| int     | 0     |
| boolean | false |
| 参照型     | null  |

## 4. static変数（クラス変数）

### ■ 定義

static修飾された変数。

```java
class User {
    static int count;
}
```

### ■ 特徴

- クラスに1つだけ存在
- メソッドエリアに保存
- クラスロード時に生成
- JVM終了まで存続

### ■ 生成タイミング

クラスが「初期化」されるとき。

例：
- newされたとき
- staticメンバが参照されたとき

## 5. スコープとは何か

スコープとは「変数が参照可能な範囲」。

### ■ ブロックスコープ

```java
if(true){
    int x = 10;
}
System.out.println(x); // エラー
```
波括弧内のみ有効。

### ■ for文スコープ

```java
for(int i = 0; i < 10; i++){
}
System.out.println(i); // エラー
```

### ■ シャドーイング

```java
class Test {
    int x = 10;

    void method() {
        int x = 20;
        System.out.println(x); // 20
    }
}
```
ローカルが優先される。

## 6. 変数のライフサイクル

| 種類     | 生成     | 破棄     | 保存場所    |
| ------ | ------ | ------ | ------- |
| ローカル   | メソッド呼出 | メソッド終了 | スタック    |
| インスタンス | new    | GC     | ヒープ     |
| static | クラス初期化 | JVM終了  | メソッドエリア |

## 7. メモリから見た変数

例：以下のコードに解説
```java
class Test {
    static int s = 1;
    int i = 2;

    void method() {
        int x = 3;
    }
}
```

### ■ メソッドエリア
```
Testクラス情報
s = 1
```
### ■ ヒープ
```
Testオブジェクト
  i = 2
```
### ■ スタック
```
x = 3
```

## 8. staticの本質

staticは「オブジェクトに属さない」。

つまり：
- インスタンス不要
- 共有データ

### ■ 実務での使い所

- 定数（static final）
- ユーティリティメソッド
- キャッシュ

## 9. 実質final（ラムダ制約）

```java
int x = 10;
Runnable r = () -> {
    System.out.println(x);
};
```
xは変更できない。

理由：
- ローカル変数はスタック上。
- ラムダはヒープに存在。

コピーが必要になるため、変更禁止。

## 10. 実務で重要なポイント

### ① static乱用問題

staticにすると：
- テスト困難
- 依存が固定化
- 並行処理で事故

### ② 参照型の共有問題
```java
List list = new ArrayList();
```

- これをstaticにすると全員共有。
  - スレッドセーフでなくなる。

### ③ メモリリークの原因

staticに大きなオブジェクトを保持すると：
- GCされない。

## 11. まとめ

- 変数は保存場所で性質が変わる
- ローカルはスタック
- インスタンスはヒープ
- staticはクラス単位
- スコープは波括弧で決まる
- staticは共有資源であり慎重に扱う
