## 結論
アクセス修飾子（しゅうしょくし）は、「クラスの責務と公開範囲」を制御するための仕組み。

## なぜ必要か
すべてを public にすると、  
どこからでも触れてしまい、変更に弱くなる。

## 種類と公開範囲
- protected : 同一パッケージ内のクラスと、異なるパッケージ内のサブクラス
- default : 同一パッケージ内のクラス
- public : 全てのクラス
- private : 同一クラス内のみ

## 実務での考え方
基本は **private → 必要に応じて開ける**。
