## 1. DIの基本概念

DI（依存性注入）とは、クラスが自分で依存オブジェクトを生成せず、
外部から渡してもらう設計手法です。

### ❌ DIなし（密結合）

``` java
class UserService {
    private UserRepository repo = new UserRepository();
}
```

### ✅ DIあり（疎結合）

``` java
class UserService {
    private final UserRepository repo;

    public UserService(UserRepository repo) {
        this.repo = repo;
    }
}
```

## 2. DIの目的

-   疎結合にする
-   テストしやすくする
-   変更に強い設計にする

## 3. DIの本質

DIは「オブジェクトの生成責任を外部に委ねる」ことです。
