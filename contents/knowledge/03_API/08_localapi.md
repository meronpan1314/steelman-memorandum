## 結論

ローカルAPIとは、**同じプログラム・同じ実行環境の中で呼び出されるAPI**のこと。</br>
Java特有の概念ではなく、**ほぼすべてのプログラミング言語に存在する共通概念**。

---

## ローカルAPIとは何か

* 同じプロセス
* 同じメモリ空間
* ネットワーク通信なし

で呼び出されるAPI。

例（Java）：

```java
List<String> list = new ArrayList<>();
list.add("A");
int size = list.size();
```

👉 呼び出しは即時で、失敗要因も限定的。

---

## リモートAPIとの違い（再確認）

| 観点   | ローカルAPI | リモートAPI    |
| ---- | ------- | ---------- |
| 実行場所 | 同一プロセス  | 別プロセス・別マシン |
| 通信   | なし      | ネットワーク     |
| 速度   | 非常に速い   | 遅延あり       |
| 失敗要因 | 少ない     | 多い         |

👉 CORBA / REST / gRPC はすべてリモートAPI。

---

## JavaにおけるローカルAPI

### 標準ライブラリAPI

* `String`
* `List`, `Map`
* `Math`
* `File`

👉 Javaが公式に提供しているAPI。

---

### 自作クラスのメソッド

```java
class Calculator {
    int add(int a, int b) {
        return a + b;
    }
}
```

👉 これも立派なローカルAPI。

---

## ローカルAPIはJava特有か？

### 結論：**いいえ**

ローカルAPIは、
**プログラミング言語共通の考え方**。

---

## 他言語の例

### Python

```python
my_list.append(1)
len(my_list)
```

### JavaScript

```javascript
array.push(1)
array.length
```

### C++

```cpp
vec.push_back(1);
vec.size();
```

👉 どれも同じ。

---

## 「API」という言葉が分かりにくくなる理由

ローカルAPIは、
**普段あまり意識されずに使われている**。

* メソッド呼び出し
* 関数呼び出し

👉 APIという言葉を使わなくても困らない。

一方、
リモートAPIは意識しないと壊れる。

---

## ローカルAPI設計の考え方

ローカルAPIでも、
設計の良し悪しは存在する。

* 名前は分かりやすいか
* 責務が明確か
* 使い方を間違えにくいか
